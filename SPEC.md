# 关联企业库存商品 ERP —— 需求规格说明书（SPEC v0.1）

> 状态：**待评审**　|　启用日期：2026-06-01　|　币种：人民币（CNY）

---

## 1. 项目概述

为三家关联企业建设一套**共用的库存商品 ERP**，覆盖采购入库、销售出库、资金与往来、票据、对账等业务。
系统采用**单一系统 + 多公司账套**架构：一套程序、一个数据库，每条单据/分录都带"所属公司"标签；三家公司各算各的账（独立期初、独立报表、互不串账），但关联交易可在系统内自动联动。

### 1.1 三家公司（关联方）
| 编号 | 公司全称 |
|---|---|
| C1 | 安博诺新材料科技（上海）有限公司 |
| C2 | 上海恒本源化学有限公司 |
| C3 | 鸿威达新材料科技（上海）有限公司 |

### 1.2 规模
- 每家公司库存品种 ≤ 100 个；人员少；当前 5 人使用。
- 单人操作为主，暂不开通"复核/审核"流程（**预留**字段与流程开关，默认关闭）。

## 2. 角色与权限

| 角色 | 可见/可做 |
|---|---|
| 总经理 | **总览表**（见 §9），跨三家公司只读汇总 |
| 出纳 | GUI 同总经理（只读总览）；+ 资金/票据相关录入 |
| 采购 | 仅查看库存商品数量；录入采购入库相关 |
| 销售 | 仅查看库存商品数量；录入销售出库相关 |
| 财务 | 发票、付款/收款、日记账、对账等 |

- **一人可兼多角色**（如同一人既是采购又是销售）。权限按"角色集合"叠加。
- 权限模型：RBAC（用户 → 角色 → 权限点）。
- 数据范围：可限定某用户只能看某公司，或可见多公司（按需配置）。

## 3. 业务流程

### 3.1 采购
```
目标主线（M18，见 §20）：
采购订单 → 采购入库单 / 采购发票 → 付款登记

过渡期（历史无订单时仍可用）：
采购入库单 → 采购发票登记 → 付款登记
```
- 采购方式：**外购**、**借调**（借调可继续不经采购订单）。
- 含其他费用录入（见 §6.2）。

### 3.2 销售
```
目标主线（M18，见 §20）：
销售订单 → 销售出库单 / 销售发票 → 收款登记

过渡期（历史无订单时仍可用）：
销售出库单 → 销售发票开具 → 收款登记
```
- 销售方式：**销售**、**归还**（归还/借出可继续不经销售订单）。
- 含其他费用录入（见 §6.2）。

## 4. 存货核算

- **计价方法：移动加权平均法**（每次入库后重算平均单价，出库按当前平均单价结转成本）。
- 库存以**数量金额式**管理（数量 + 金额 + 单价）。

### 4.1 借调 / 归还（无发票、不涉税）
- **借调入库**：按**约定成本（暂估单价）**入库，数量、金额均进库存；同时挂一笔 **"借调往来"**（性质类似其他应付）；**不产生增值税**。
- **归还出库**：冲减库存，并冲减对应的"借调往来"。
- 借调/归还在关联企业间**自动联动**（见 §5）。

## 5. 关联交易联动

当一家公司录入面向另一家关联公司的交易时，对方账套**完全自动**生成镜像单据（**无需对方确认**）：

| 录入方动作 | 自动为对方生成 |
|---|---|
| A 对 B **销售出库** | B 的**采购入库**（外购） |
| A 对 B **借调出库**（借出/归还出库） | B 的**借调入库** |

- 自动生成的对方单据：**价格、数量直接照搬**源单据。
- 联动的撤销：源单据作废时，联动生成的镜像单据一并作废。

### 5.1 M4 实现决议（2026-06-05 自主确定）
- **触发**：销售出库单的客户其 `related_company` 指向系统内关联公司 B 时，保存出库即在 **B 账套自动生成一张采购入库单（外购）**，无需对方确认。
- **照搬规则**：逐行照搬商品（按编码在 B 账套匹配，**B 无则自动建**，复制名称/规格/单位/税率）、数量。计价口径（**2026-06-26 修正**）：
  - **销售镜像（外购）**：B 的入库成本 = **源出库行的不含税售额**（B 是按 A 的售价买进，含税/税额一并镜像，便于 B 后续按此额收采购发票）；若该行未录售价（不含税额为 0）则回退按移动加权结转成本平移。
  - **借调镜像（借调入库）**：按 A 的移动加权结转成本平移（借调不涉税、无加价）。
  - 旧规则曾一律按结转成本平移（因当时出库单无售价，M7 后已有三价，故销售改按售价镜像）。
- **互链**：源出库单与镜像入库单互相记录对方 ID（source/mirror），便于追溯与联动作废。
- **作废联动**：源出库单作废 → 自动作废其镜像采购入库单。作废按**反向冲减**恢复库存（数量、金额、移动加权重算）；若作废会导致任一商品库存为负则**拒绝作废**（提示先冲销后续业务）。
- **借调联动**（借调出库↔借调入库）随 **M6 借调单据**一并实现；本期 M4 先做销售↔采购镜像。

## 6. 税 与 费用

### 6.1 增值税
- 所有采购/销售发票**默认税率 13%**，**允许按行手工改**（如 9% / 6% / 0%）。
- 金额区分**含税金额**与**不含税金额**，二者按税率自动换算。

### 6.2 其他费用
- 采购入库、销售出库中均可录入"其他费用"。
- 费用类别**可自行增加**（如运费、差旅费…）。
- 每个费用类别设 **"是否计入存货成本"** 开关：
  - 计入成本（如运费）→ 抬高相关存货单价；
  - 不计入（如差旅费）→ 作期间费用单列。

## 7. 资金与往来

### 7.1 银行存款日记账
- 来源一：采购"**付款登记**"、销售"**收款登记**"时，**系统自动生成**一条银行存款日记账（不重复录入）。
- 来源二：支持 **Excel 增量导入**（如网银流水）与 **Excel 导出**。
- **冲销（核销）**：把已登记的收/付款，**手工点击**与对应的**应收账款 / 应付账款**做核销。

### 7.2 应收票据 / 应付票据日记账
- 支持 **Excel 导入 / 导出**。
- **应收票据冲销**：收到的银行承兑汇票**背书转让给供应商抵付应付账款**（手工选择）；亦可冲销应收。
- **应付票据冲销**：开票据给供应商抵减应付账款（手工选择）。

### 7.4 M3 票据设计决议（2026-06-05 自主确定）
- **应收票据 / 应付票据**各为一张票据单据：金额、票号、出票日、到期日、对方单位、状态（在手 / 已背书 / 已结算 / 作废）。
- **应收票据**两类用途（可分次、部分使用、一票核多张发票）：① **核销应收账款**（冲销售发票）；② **背书转让给供应商抵付应付账款**（冲采购发票），背书后状态置「已背书」。
- **应付票据**：开给供应商**抵减应付账款**（冲采购发票）。
- 票据以「金额 − 已用额 = 未用额」跟踪；冲销不得超过票据未用额，也不得超过发票未核销额。
- **Excel 导入 / 导出**票据台账（与银行日记账同套思路）。
- 票据到期托收 / 兑付（进银行存款）本期不建模，状态预留；本系统仅跟踪票据余额（SPEC §9.1 总览含应收票据）。

### 7.3 M2 资金往来设计决议（2026-06-05 确认）
- **往来产生时点**：应付/应收账款由**发票登记**产生 —— 采购入库 / 销售出库本身只增减库存、**不挂往来**（借调除外，借调走"借调往来"见 §4.1）。
- **发票录入两种方式**：① 关联一张或多张入库/出库单，带出商品与不含税金额；② 独立录入（仅填往来对象 + 不含税金额）。均**按行税率算含税**（默认 13% 可改，含税/不含税自动换算，见 §6.1）。
- **发票即往来单据**：发票含税总额 = 应付/应收原始金额；发票上记"已核销额"，未核销额 = 原始 − 已核销。
- **核销分两步**：付款 / 收款先登记（**自动生成银行存款日记账**），再到核销界面**手工**把款项与发票勾对；**支持部分核销**、一款核多票、一票多款。
- **银行账户**：每公司可建**多个银行账户**（基本户/一般户等）；银行存款日记账按账户记。
- **账务范围**：本系统跟踪 **应付 / 应收 / 银行存款 / 借调往来**，以及 §7.5 往来对冲后仍由发票体现的应收应付净变动、§7.6 票据拆借产生的 **其他应收·关联方 / 其他应付·关联方** 台账余额；**不做完整借贷总账**。增值税额记发票备查，应付/应收按**含税额**挂账；其他应收/其他应付台账按票面或拆借额（不含税约定）记账，将来导出凭证时映射科目（见 §7.5 / §7.6）。
- **与库存成本关系**：库存成本在入库（M1）即按不含税单价结转，**发票不回改库存**；发票不含税额与入库金额不符时仅提示、不强制相等。

### 7.5 往来对冲（应收↔应付互抵）
同一账套内，当某一外部单位（或同一关联企业）**既是客户又是供应商**时，可不经资金收付，将未结**应收账款**与未结**应付账款**按约定金额互抵。

- **菜单**：财务管理 → 往来对冲（列表 / 新建 / 撤销）。
- **对象**：须同时选定本账套下的 **客户** 与 **供应商**（名称可不强制相同；内部关联户建议客户、供应商均挂同一 `related_company`）。
- **操作**：勾选一张或多张未结清销售发票 + 一张或多张未结清采购发票；对冲合计应付侧 = 对冲合计应收侧；**支持部分对冲**。
- **账务效果**（同一事务）：销售发票 `settled_amount` 增加、采购发票 `settled_amount` 增加；**不产生**银行存款日记账。留独立对冲单据号与明细行，审计记 `OFFSET`。
- **边界**：不作废发票本身；已对冲部分不可再重复核销；撤销对冲则回退双方 `settled_amount` 并删除对冲单。期初发票允许参与（与付款核销规则一致）。
- **不等于**：付款/收款核销、票据背书抵付、货物借调往来、笼统「手工改余额」。
- **将来凭证映射**（本系统不生成凭证，仅约定）：`借：应付账款 / 贷：应收账款`（金额=对冲额，按含税）。

### 7.6 关联公司票据拆借
三家关联企业之间，可将本账套持有的**应收票据**或己方开出的**应付票据**拆借给另一家关联公司（资金性票权调剂），**不**用于抵供应商货款。

- **菜单**：财务管理 → 票据拆借（列表 / 拆出 / 归还 / 查询）。
- **与现有票据能力的区别**：
  | | 背书抵付（§7.2 / §7.4） | 票据拆借（本节） |
  |---|---|---|
  | 对手 | 供应商（货款） | 系统内关联公司（`related_company` / 目标账套） |
  | 效果 | 冲减应付账款（或冲应收） | 出借方挂其他应收·关联方；借入方挂其他应付·关联方，并转移票面持有 |
  | 联动 | 本账套内 | **两侧账套同一事务镜像** |
- **应收票据拆借（A→B）**：
  - A：扣减该票未用额（视为票面转出）；登记「票据拆借」单据；增加对 B 的 **其他应收·关联方**。
  - B：登记对称拆借单；增加持有应收票据（新建或镜像票号/票面要素）；增加对 A 的 **其他应付·关联方**。
- **应付票据拆借**：对称处理——出借方减少己方应付票据负担并挂其他应收；借入方承接应付票据并挂其他应付（细则实现时与应付票状态机对齐）。
- **归还**：借入方将票权归还出借方，冲减两侧其他应收/其他应付，票面回到出借方；允许部分归还。
- **禁止混淆**：货物借调往来（§4.1）不走本节；背书抵付不得记入其他应收/其他应付。
- **报表**：其他应收·关联方 / 其他应付·关联方余额（按对手公司）可进账户余额表或独立查询；票据余额表反映拆借后的持有变化。
- **将来凭证映射**（约定）：出借方 `借：其他应收款-关联方 / 贷：应收票据`（或应付票据相关科目）；借入方 `借：应收票据 / 贷：其他应付款-关联方`。本系统本期只记台账与单据，**不生成总账凭证**。

## 8. 期初数据 与 对账

### 8.1 期初（启用日 2026-06-01）
三家公司各自的以下期初，用 **Excel 模板批量导入**：
- 期初库存（数量、金额）
- 期初应收账款 / 应付账款
- 期初银行存款余额
- 期初应收票据 / 应付票据
- **期初发出商品**（已出库未开票的结转成本；不重复减库存）
- **期初应付账款-暂估**（已入库未收票的不含税金额；不重复加库存）

### 8.2 月底对账
对以下项目进行核对（系统出对账表，标出差异）：
- 银行存款
- 应收票据
- 商品数据（数量金额）
- 应收账款金额
- 应付账款金额

## 9. 界面与报表

- **主界面 GUI**，按角色显示不同内容。
- **电脑端 + 手机端**：响应式网页（同一网址，手机自动适配），不单独做 App。

### 9.1 总览表（总经理 / 出纳）
跨三家公司展示，每项均含 **期初 / 本期收入 / 本期发出 / 期末结存**：
- 各公司银行存款
- 库存商品（**数量金额式**）
- 发出商品（出库成本，开票冲减）
- 供应商往来（应付）
- 应付账款-暂估（入库不含税未收票）
- 客户往来（应收）
- 应收票据

### 9.2 其他角色
- 销售、采购：仅"库存商品数量"。

## 10. 技术选型（建议，待确认）

| 层 | 选型 | 理由 |
|---|---|---|
| 后端 | **Python + Django** | 自带用户/权限/后台/ORM，表单与 Excel 处理方便，适合财务类数据系统 |

| 部署 | **公司 Windows 主机**，Waitress 运行；**花生壳内网穿透**支持外网手机/电脑访问（强制 HTTPS）；详见 `DEPLOY.md` | A1 + 外网访问需求 |

> 关键原则：涉及金额/库存的操作必须在**数据库事务**内完成，保证一致性；联动、冲销、结转都要可追溯（留操作日志）。

## 11. 里程碑（垂直切片，先打通一条线）

> 思路：不横向铺摊子，先竖着打通"一条完整业务线"验证骨架，再逐模块扩展。

- **M0 地基**：项目骨架、登录、三公司账套、角色权限、商品/客户/供应商基础资料。
- **M1 垂直切片（核心验证）**：单公司"**采购入库 → 库存增加（移动加权平均）→ 销售出库 → 库存减少**"端到端打通 + 库存数量金额报表。
- **M2 资金往来**：采购发票/付款、销售发票/收款；银行存款日记账（自动生成 + Excel 导入导出）+ 冲销核销。
- **M3 票据**：应收/应付票据日记账 + Excel + 冲销。
- **M4 关联联动**：销售↔采购、借调↔借入 跨公司自动镜像。
- **M5 期初 + 对账 + 总览表**：Excel 期初导入、月底对账、总经理总览表。
- **M6 其他费用、税率可改、借调往来** 等细化。

## 20. M18 销售/采购订单主线（2026-07-15 确认）

> 目标：以**销售订单 / 采购订单**为业务主单据；出库/入库、发票为执行单据。  
> 兼容：6 月及以前**已完成**业务不强制补订单；**未完成**业务可补订单并回挂，**不改已入账金额与库存成本**。

### 20.1 设计原则
1. **订单是业务主线**（Business Master Document）；出库/入库、发票、收付款均为执行或结算单据。
2. **按订单行关联与统计**（非仅表头），天然支持部分发货/收货、多次开票、拆并票。
3. **出库 ↔ 发票、入库 ↔ 发票的可选行关联保留**（`source_outbound_line` / `source_inbound_line`），用于发出商品、应付暂估、成本与售价勾稽；**不得**为上订单而砍掉该匹配。
4. **收款默认仍核销销售发票、付款默认仍核销采购发票**；订单仅汇总已收/已付（预收预付后续可挂订单，但转应收应付仍落实到发票）。
5. **会计时点保持现状（模式 A）**：  
   - 出库/入库：动库存（销售出库进发出商品；采购入库进暂估口径）；关联交易镜像仍由出库触发。  
   - 开票：挂应收/应付；开票冲减发出商品 / 暂估。  
   - **不采纳**「开票同时结转主营业务成本、出库暂不结成本」的模式 B（与当前总览口径冲突，本里程碑不做）。
6. **有订单则数量受控；无订单则保持现状**（校验不得阻断历史无来源单据）。

### 20.2 目标单据关系
```
销售订单（行）
   ├─→ 销售出库单（行可引用订单行）
   └─→ 销售发票（行可引用订单行；有出库时可同时挂出库行）
         └─→ 收款核销发票

采购订单（行）
   ├─→ 采购入库单（行可引用订单行）
   └─→ 采购发票（行可引用订单行；有入库时可同时挂入库行）
         └─→ 付款核销发票
```
- 录入订单后，可由订单**生成**出库/入库、发票（带出商品、数量、单价/金额、税率等，可改）。
- 一张订单可多次生成出库（入库）、多次生成发票；多张出库可合并开一张发票；一张出库可拆多张发票——均以**订单行剩余数量**为上限（有订单时）。

### 20.3 订单行统计（系统自动算，不手工维护）
**数量（销售 / 采购对称）**
- 订单数量
- 累计发货数量 / 累计收货数量
- 累计开票数量
- 待发货 / 待收货 = 订单数量 − 累计发货/收货
- 待开票 = 订单数量 − 累计开票数量  
  （是否允许「开票数量 > 发货数量」：允许，以支持先票后货；待发货与待开票独立。）

**金额（销售示意；采购对称为合同额/已收票/已付款）**
- 合同金额（订单行含税或约定口径，实现时与现有三价一致）
- 已开票金额、未开票金额
- 已收款/已付款金额、未收款/未付款金额（按关联发票核销额汇总到订单）

### 20.4 订单状态（表头可由行汇总）
| 维度 | 状态 |
|---|---|
| 发货/收货 | 未发货(未收货) / 部分 / 全部 |
| 开票 | 未开票 / 部分 / 全部 |
| 收付款 | 未收款(未付款) / 部分 / 全部 |

（收入确认状态预留，本期不做。）

### 20.5 来源追溯
- 出库/入库/发票表头或行可记：`source_type`（如 SO / PO）、`source_no`、以及**订单行外键**（优先）。
- 历史单据允许来源为空。

### 20.6 先票后货 / 先货后票
- **先货后票**：订单 → 出库（库存↓、发出商品↑）→ 发票（应收↑、发出商品↓）。
- **先票后货**：订单 → 发票（应收↑；发出商品不变）→ 出库（库存↓、发出商品↑；若发票已挂订单行，出库后再按匹配冲减发出商品——与现口径一致：发出商品增在出库、减在关联出库的开票）。
- 独立发票（不挂出库行）仍允许；此时发出商品不因该发票减少（与现状一致）。

### 20.7 历史数据与补单（重要）
| 情形 | 处理 |
|---|---|
| **已完成业务**（建议判定见下） | **不补**订单；出库/发票来源为空；报表与核销继续走旧链路 |
| **未完成业务** | 可新建订单 → **回挂**已有未结出库/入库行、未结发票行 → 按已有数量**回写**累计发货/开票 → 此后新发生从订单生成 |
| 补单规则 | **仅补关联与进度字段，不改库存成本、发票金额、核销额、银行与票据已入账数** |

**「已完成」建议判定（实现默认；可在补单工具中人工勾选例外）**
- **销售**：相关出库已处理完毕，且销售发票未核销额=0（或业务确认不再追收）；无「已出库未开票」残留，或残留已决定不再开票并关闭。
- **采购**：对称（入库/收票/付款）。
- **6 月已入账且已结清的闭环业务**：默认视为已完成，不批量造空订单。

### 20.8 异常监控（订单上线后报表可逐步增加）
1. 已发货未开票　2. 已开票未发货　3. 已开票未收款　4. 已发货未收款　5. 已收款未开票（预收）  
与现有「已出库未开票 / 已入库未收票」明细并存；订单维度为增量，不替代成本类明细。

### 20.9 范围与非目标（本里程碑）
- **做**：销售订单 + 采购订单模型与录入；由订单生成出库/入库与发票；行级累计与状态；历史兼容与补单回挂工具（可分步）。
- **不做（本期）**：砍掉出库↔发票关联；开票确认收入并结转成本（模式 B）；强制全部历史回补订单；应付/应收脱离发票改挂订单；借调/归还强制走订单。
