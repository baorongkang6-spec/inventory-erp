# 关联企业库存商品 ERP —— 需求规格说明书（SPEC v0.1）

> 状态：**待评审**　|　启用日期：2026-06-01　|　币种：人民币（CNY）

---

## 1. 项目概述

为三家关联企业建设一套**共用的库存商品 ERP**，覆盖采购入库、销售出库、资金与往来、票据、对账等业务。
系统采用**单一系统 + 多公司账套**架构：一套程序、一个数据库，每条单据/分录都带"所属公司"标签；三家公司各算各的账（独立期初、独立报表、互不串账），但关联交易可在系统内自动联动。

### 1.1 三家公司（关联方）
| 编号 | 公司全称 |
|---|---|
| C1 | 安博诺新材料科技（上海）有限公司 |
| C2 | 上海恒本源化学有限公司 |
| C3 | 鸿威达新材料科技（上海）有限公司 |

### 1.2 规模
- 每家公司库存品种 ≤ 100 个；人员少；当前 5 人使用。
- 单人操作为主，暂不开通"复核/审核"流程（**预留**字段与流程开关，默认关闭）。

## 2. 角色与权限

| 角色 | 可见/可做 |
|---|---|
| 总经理 | **总览表**（见 §9），跨三家公司只读汇总 |
| 出纳 | GUI 同总经理（只读总览）；+ 资金/票据相关录入 |
| 采购 | 仅查看库存商品数量；录入采购入库相关 |
| 销售 | 仅查看库存商品数量；录入销售出库相关 |
| 财务 | 发票、付款/收款、日记账、对账等 |

- **一人可兼多角色**（如同一人既是采购又是销售）。权限按"角色集合"叠加。
- 权限模型：RBAC（用户 → 角色 → 权限点）。
- 数据范围：可限定某用户只能看某公司，或可见多公司（按需配置）。

## 3. 业务流程

### 3.1 采购入库
```
采购入库单 → 采购发票登记 → 付款登记
```
- 采购方式：**外购**、**借调**。
- 含其他费用录入（见 §6.2）。

### 3.2 销售出库
```
销售出库单 → 销售发票开具 → 收款登记
```
- 销售方式：**销售**、**归还**。
- 含其他费用录入（见 §6.2）。

---

## 4. 存货核算

- **计价方法：移动加权平均法**（每次入库后重算平均单价，出库按当前平均单价结转成本）。
- 库存以**数量金额式**管理（数量 + 金额 + 单价）。

### 4.1 借调 / 归还（无发票、不涉税）
- **借调入库**：按**约定成本（暂估单价）**入库，数量、金额均进库存；同时挂一笔 **"借调往来"**（性质类似其他应付）；**不产生增值税**。
- **归还出库**：冲减库存，并冲减对应的"借调往来"。
- 借调/归还在关联企业间**自动联动**（见 §5）。

---

## 5. 关联交易联动

当一家公司录入面向另一家关联公司的交易时，对方账套**完全自动**生成镜像单据（**无需对方确认**）：

| 录入方动作 | 自动为对方生成 |
|---|---|
| A 对 B **销售出库** | B 的**采购入库**（外购） |
| A 对 B **借调出库**（借出/归还出库） | B 的**借调入库** |

- 自动生成的对方单据：**价格、数量直接照搬**源单据。
- 联动的撤销：源单据作废时，联动生成的镜像单据一并作废。

### 5.1 M4 实现决议（2026-06-05 自主确定）
- **触发**：销售出库单的客户其 `related_company` 指向系统内关联公司 B 时，保存出库即在 **B 账套自动生成一张采购入库单（外购）**，无需对方确认。
- **照搬规则**：逐行照搬商品（按编码在 B 账套匹配，**B 无则自动建**，复制名称/规格/单位/税率）、数量；**单价采用源出库行的移动加权结转成本**作为 B 的入库成本单价（出库单上无售价，售价在发票侧；关联调拨按成本平移）。
- **互链**：源出库单与镜像入库单互相记录对方 ID（source/mirror），便于追溯与联动作废。
- **作废联动**：源出库单作废 → 自动作废其镜像采购入库单。作废按**反向冲减**恢复库存（数量、金额、移动加权重算）；若作废会导致任一商品库存为负则**拒绝作废**（提示先冲销后续业务）。
- **借调联动**（借调出库↔借调入库）随 **M6 借调单据**一并实现；本期 M4 先做销售↔采购镜像。

## 6. 税 与 费用

### 6.1 增值税
- 所有采购/销售发票**默认税率 13%**，**允许按行手工改**（如 9% / 6% / 0%）。
- 金额区分**含税金额**与**不含税金额**，二者按税率自动换算。

### 6.2 其他费用
- 采购入库、销售出库中均可录入"其他费用"。
- 费用类别**可自行增加**（如运费、差旅费…）。
- 每个费用类别设 **"是否计入存货成本"** 开关：
  - 计入成本（如运费）→ 抬高相关存货单价；
  - 不计入（如差旅费）→ 作期间费用单列。

## 7. 资金与往来

### 7.1 银行存款日记账
- 来源一：采购"**付款登记**"、销售"**收款登记**"时，**系统自动生成**一条银行存款日记账（不重复录入）。
- 来源二：支持 **Excel 增量导入**（如网银流水）与 **Excel 导出**。
- **冲销（核销）**：把已登记的收/付款，**手工点击**与对应的**应收账款 / 应付账款**做核销。

### 7.2 应收票据 / 应付票据日记账
- 支持 **Excel 导入 / 导出**。
- **应收票据冲销**：收到的银行承兑汇票**背书转让给供应商抵付应付账款**（手工选择）；亦可冲销应收。
- **应付票据冲销**：开票据给供应商抵减应付账款（手工选择）。

### 7.4 M3 票据设计决议（2026-06-05 自主确定）
- **应收票据 / 应付票据**各为一张票据单据：金额、票号、出票日、到期日、对方单位、状态（在手 / 已背书 / 已结算 / 作废）。
- **应收票据**两类用途（可分次、部分使用、一票核多张发票）：① **核销应收账款**（冲销售发票）；② **背书转让给供应商抵付应付账款**（冲采购发票），背书后状态置「已背书」。
- **应付票据**：开给供应商**抵减应付账款**（冲采购发票）。
- 票据以「金额 − 已用额 = 未用额」跟踪；冲销不得超过票据未用额，也不得超过发票未核销额。
- **Excel 导入 / 导出**票据台账（与银行日记账同套思路）。
- 票据到期托收 / 兑付（进银行存款）本期不建模，状态预留；本系统仅跟踪票据余额（SPEC §9.1 总览含应收票据）。

### 7.3 M2 资金往来设计决议（2026-06-05 确认）
- **往来产生时点**：应付/应收账款由**发票登记**产生 —— 采购入库 / 销售出库本身只增减库存、**不挂往来**（借调除外，借调走"借调往来"见 §4.1）。
- **发票录入两种方式**：① 关联一张或多张入库/出库单，带出商品与不含税金额；② 独立录入（仅填往来对象 + 不含税金额）。均**按行税率算含税**（默认 13% 可改，含税/不含税自动换算，见 §6.1）。
- **发票即往来单据**：发票含税总额 = 应付/应收原始金额；发票上记"已核销额"，未核销额 = 原始 − 已核销。
- **核销分两步**：付款 / 收款先登记（**自动生成银行存款日记账**），再到核销界面**手工**把款项与发票勾对；**支持部分核销**、一款核多票、一票多款。
- **银行账户**：每公司可建**多个银行账户**（基本户/一般户等）；银行存款日记账按账户记。
- **账务范围**：本系统只跟踪 **应付 / 应收 / 银行存款** 余额，**不做完整借贷总账**；增值税额记发票备查，应付/应收按**含税额**挂账。
- **与库存成本关系**：库存成本在入库（M1）即按不含税单价结转，**发票不回改库存**；发票不含税额与入库金额不符时仅提示、不强制相等。

## 15. M8 银行收支与对账（2026-06-06 讨论确认：混合流）
客户希望**银行存款日记账成为完整资金账本**（含费用/税/工资/利息/内部划转等，非仅供应商客户收付）。三种流程讨论后定为**混合流**：
- **往来收付仍在源头登记**：客户收款、供应商付款继续走收款/付款登记（手工选对象 + 核销发票），自动生成日记账行——因为"哪个对象、核销哪张发票"的业务意图只有人录得准，网银摘要推不出（尤其部分核销、一笔对多张、关联对冲），且关联镜像(M4)依赖单据带明确往来对象。
- **非往来收支直接录日记账**：新增「其他收支登记」入口（手工或 Excel），按业务类型分类，不挂 AR/AP。
- **Excel 导入定位为对账工具**：导入整月网银流水，与已登记日记账行按 流水号/金额/日期 自动勾对；已匹配标记已对账，未匹配的提示——是漏录费用（就地建一条其他收支）还是漏登往来（提示去补登收付款）。即"部分自动联动 + 手工确认"发生在**对账层**，而非靠导入猜发票。

实现切片：
- **M8-1 业务类型**：`BankJournal` 加 `entry_type`（往来结算/费用/税费/工资/内部划转/其他）+ 迁移回填（Payment/Receipt 来源标"往来结算"，导入历史标"其他"）；日记账报表加类型列与筛选。
- **M8-2 其他收支登记**：手工录非往来收付（方向/金额/对方/类型/费用类别/备注/流水号）生成日记账；列表 + 筛选 + 导出 + 作废。
- **M8-3 导入对账**：网银流水与已登记行勾对、已匹配/未匹配清单、未匹配一键补录。
- **M8-4 验证与提交**。

### 15.1 实现确认（M8-1~M8-4 已完成）
M8-1 `BankJournal.entry_type`（往来结算/费用/税费/工资/内部划转/其他）+ 迁移回填；日记账报表加类型列、类型筛选（余额仍按全部流水累计）、本期合计 ✅　M8-2 「其他收支登记」`create_other_cashflow`（拒绝往来结算类型）+ 手工行可删（仅 Other）✅　M8-3 「银行对账」`reconcile_bank_journal`（流水号→日期+金额+方向勾对、置 reconciled + `BankReconcileBatch` 留痕）+ 三类清单 + 仅网银有一键建其他收支 ✅　M8-4 端到端验证（85 测试全过、对账/其他收支/类型筛选页冒烟 200）✅

## 16. M9 总览两级下钻（2026-06-06，沿用银行存款两级模式）
客户要求总览各类目都像银行存款那样**两级下钻**：类目-公司 → 该公司各分项的 期初/本期收入/本期发出/期末（保留日期窗口、当期无发生也显示期末）→ 再点某分项进其明细账（按改动后期间显示流水）。

实现确认（M9-1~M9-5 已完成）：
- **M9-1 库存商品** ✅：库存商品余额表(各商品 期初/本期/期末·金额+期末数量) → 商品流水台账(明细账，日期区间+期初/期末结存)。
- **M9-2 供应商应付** ✅：应付余额表(各供应商 期初/本期增/本期减/期末) → 供应商往来明细账(发票增、付款核销/票据抵付减、滚动余额)。
- **M9-3 客户应收** ✅：镜像 M9-2(应收余额表 → 客户往来明细账)。
- **M9-4 应收票据** ✅：应收票据余额表(各票据 期初/本期出票/本期使用/期末未用) → 票据使用明细(出票增、冲应收/背书抵应付减、滚动未用额)。
- **M9-5 验证与提交** ✅：总览五类目下钻链接齐全、各页/导出冒烟、92 测试全过。
- 全程 `?company=` 下钻不改当前账套；各余额表/明细账均支持 Excel 导出。
