# 项目：关联企业库存商品 ERP

## 这是什么
为三家关联公司（安博诺 C1 / 恒本源 C2 / 鸿威达 C3）建设的库存商品 ERP。
**单一系统 + 多公司账套**：一套程序一个库，每条数据带"所属公司"，三家各算各账、关联交易自动联动。

## 第一信息源
**`SPEC.md` 是唯一需求来源。动手前必读。** 需求变更先改 SPEC，再改代码。

## 关键约束（最容易出错的地方）
- 涉及金额/库存的操作必须在**数据库事务**内完成。
- 库存计价：**移动加权平均**。
- 增值税默认 13% 可按行改；含税/不含税自动换算。
- 借调/归还不涉税，走"借调往来"，且跨公司自动联动。
- 付款/收款登记**自动生成**银行存款日记账，别让用户重复录。
- 冲销 = 把收/付款与应收/应付**核销**；票据冲销 = 背书抵付。

## 技术栈（见 SPEC §10）
Python + Django ＋ Bootstrap5（响应式）＋ PostgreSQL（开发期可 SQLite）。

## 开发节奏
按 SPEC §11 的 M0→M6 里程碑，**垂直切片优先**：先打通 M1 的"采购入库→库存→销售出库"一条线。
每个里程碑做完即可运行、可演示、可验证。
