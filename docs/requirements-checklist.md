# ERP 需求深挖清单（可复用模板）

> 用途：**每开发一个新模块（或新单据）前，照着这张表过一遍**，系统地把需求挖全，而不是靠临场回忆。
> 挖到的新需求 → **回写进 SPEC.md**；一时定不了的 → 丢进文末"待定问题清单"，不阻塞开发。
> 本清单可随项目沉淀为"ERP 起步模板"的一部分，下个项目继续用。

---

## 用法（3 步）
1. 选定要做的模块（如"采购入库"）。
2. 逐节回答下面 A~F 的问题，能答的写进 SPEC，答不了的进"待定清单"。
3. 再让 AI 动手写代码。

---

## A. 单据生命周期（每种单据都问一遍）
- [ ] **新建**：谁能建？必填哪些字段？默认值是什么？
- [ ] **修改**：已保存的单能改吗？已审核/已付款的还能改吗？改了会影响哪些已生成的账？
- [ ] **审核/提交**：要不要审核环节？（本项目预留不开通，但字段留好）
- [ ] **作废 / 红冲**：怎么撤销一张错单？是删除、作废、还是红字冲销？冲销后库存/往来/联动单怎么连带处理？
- [ ] **查询/筛选**：按什么查（日期、公司、客户、商品、状态）？
- [ ] **打印/导出**：要不要打印单据？导出 Excel？
- [ ] **联动影响**：这张单一存，自动牵动了哪些库存、往来、对方公司单据、报表数字？

## B. 异常与边界（⚠️ 需求 80% 藏在这里）
- [ ] 退货 / 退款怎么处理？
- [ ] **负库存**允许吗？（没货也能出库吗）
- [ ] 数量/金额为 0 或负数，允许吗？
- [ ] **部分收款 / 部分付款**？多付、少付、预付怎么记？
- [ ] **跨月 / 跨期**：上月的单这月才录？月结后还能改上月吗？
- [ ] 一张单对多张发票、一张发票对多张单（多对多）？
- [ ] 借调的货被卖掉了 / 一直没归还，怎么办？
- [ ] 关联联动失败/对方单被改了，怎么对齐？
- [ ] 同一时刻两人改同一条数据（并发）怎么办？

## C. 报表倒推（从结果反推录入要求）
- [ ] 报表上**每一个数字**：从哪来、怎么算？
- [ ] 勾稽关系：**期初 + 本期收入 − 本期发出 = 期末**，能不能永远对平？
- [ ] 口径：含税还是不含税？数量金额式？按公司/按商品/按客户哪个维度汇总？
- [ ] 时间口径：按单据日期还是录入日期？月末时点怎么截？

## D. 角色走查（站在每个人的一天）
- [ ] 这个模块，**总经理 / 出纳 / 采购 / 销售 / 财务** 各自能看什么、能做什么？
- [ ] 谁只读、谁能改、谁能删？
- [ ] 一人兼多角色时，权限怎么叠加？

## E. 数据 / 字段层面
- [ ] 每个字段：**必填？唯一？默认值？格式校验？**
- [ ] **金额精度**：保留几位小数？四舍五入规则？含税/不含税换算的进位？
- [ ] 编号规则：单据号怎么生成（前缀+公司+日期+流水）？要不要按公司唯一？
- [ ] 删除是真删还是软删（留痕）？审计日志记不记？

## F. 现有资料作为需求来源（最高效）
- [ ] 有没有**现在手工在用的 Excel / 报表 / 凭证**？→ 直接拿给 AI 看，它就是最真实的需求。
- [ ] 找 **5~10 笔真实业务**（采购、销售、借调、收付款各几笔），端到端走一遍，逼出隐藏规则。

---

## 📋 ERP 专属高频陷阱（财务/库存特有，务必逐条确认）
- [ ] **计价方法**对每种出库都成立吗？（移动加权：借调/归还/退货时单价怎么算）
- [ ] **红字冲销**对库存成本、往来余额、税额、联动单的连锁影响
- [ ] **往来核销**：部分核销、跨单核销、核销后又作废，余额对得上吗？
- [ ] **税**：默认 13% 可改，含税↔不含税换算的四舍五入差异谁兜底？
- [ ] **跨期/月结**：结账后能否回改、如何调整
- [ ] **期初**导入与"本期发生"不能重复计、不能错位
- [ ] **关联联动**：作废源单 → 镜像单连带；改源单 → 镜像单同步
